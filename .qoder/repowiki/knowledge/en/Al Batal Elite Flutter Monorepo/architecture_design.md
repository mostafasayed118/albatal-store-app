The root `pubspec.yaml` declares the shared Flutter dependency graph (bloc, go_router, supabase_flutter, get_it, intl) consumed by every child module. The `flutter_app` module is the application entry point; `platform_targets` provides the Android/iOS/Linux/macOS/Windows/Web host shells that bootstrap it; `tests` depends on the same `pubspec.yaml` to run widget/Cubit/integration suites against the built app; `scripts` owns the Supabase schema migrations and Edge Functions that the app calls at runtime; `docs` records the cross-cutting contracts (DESIGN.md, INSTRUCTIONS.md, release playbooks) that both human and AI contributors follow. Cross-child wiring happens through: (1) the single `pubspec.yaml` + `analysis_options.yaml` for linting, (2) `.env.example` plus `l10n.yaml` for environment and localization configuration, and (3) the `supabase/` directory of SQL migrations and Edge Function sources referenced by both the app's runtime code and the `scripts/` deploy tooling.
