Flat collection of human-readable Markdown guides consumed by developers and QA rather than generated artifacts. Files split into three roles: (1) pre-release gates — `release-readiness.md` enumerates code quality, database, payments, auth, admin, notifications, monitoring, localization, environment, and documentation checks plus a numbered pre-launch manual runbook; (2) verification scripts — `staging-verification.md`, `acceptance-checklist.md`, and `supabase-integration.md` provide step-by-step test matrices and setup instructions against the staging Supabase project; (3) learning walkthroughs — `foundation-walkthrough.md`, `money-walkthrough.md`, and `storefront-walkthrough.md` explain architectural slices (settings/theme/locale foundation, Money value object, storefront architecture) with ownership lists, tests, limitations, and self-check questions. There is no build tooling or code generation tied to this directory; it is referenced from `release-readiness.md` as part of the documentation checklist.
