Flat `test/` directory organized by feature rather than by layer: per-cubit unit tests (`*_cubit_test.dart`) exercise domain state transitions with hand-rolled repository stubs; per-page/widget tests (`*_page_test.dart`, `address_form_test.dart`, etc.) pump a minimal `MaterialApp` with `BlocProvider` wiring in-memory persistence (`MemoryStorefrontPersistence`) and local repositories; cross-cutting concerns live in dedicated files — `auth_test.dart` covers pure value-object behavior (`AuthState`, `Profile.copyWith`), `accessibility_test.dart` asserts tooltips and RTL directionality under both `en` and `ar` locales, `contrast_audit_test.dart` runs color contrast checks, and `integration_test.dart` drives the Orders page through a full order placement. Payment flow is split into `payment_checkout_flow_test.dart` (state machine via a broadcast `StreamController`-backed `_FlowStub`), `payment_integration_test.dart`, `payment_navigation_test.dart`, and `paymob_url_guard_test.dart`. Global test bootstrap lives in `widget_test.dart` which initializes `SharedPreferences.setMockInitialValues`, a fake Supabase client, and the DI container before running the smoke test that boots `AlBatalApp`.
