The app follows a feature-sliced architecture under `lib/`:
- `main.dart` bootstraps the process: initializes logging (`Log.setLevel`), sets `Bloc.observer = AppBlocObserver()`, then tries to initialize Supabase via `SupabaseConfig.initialize()` and DI via `configureDependencies()`; on failure it renders `_BootstrapErrorApp` instead of crashing.
- `app.dart` defines `AlBatalApp`, which wires all Cubits (Settings, Catalog, Cart, Wishlist, Orders, Addresses, Auth, Admin) through `MultiBlocProvider`, resolves repositories from `getIt`, and wraps `MaterialApp.router` with theme/locale-aware `BlocBuilder<SettingsCubit>`.
- Dependency resolution is centralized in `shared/services/service_locator.dart`: `GetIt.instance` registers repository interfaces against their data-layer implementations (local `SharedPreferences`-backed or Supabase-backed) as lazy singletons; Cubits are intentionally NOT registered here and are created per-route by `BlocProvider`.
- Routing is declarative in `shared/routing/app_router.dart` using `go_router`: a `ShellRoute` groups the bottom-nav shell routes (`/home`, `/categories`, `/catalog`, `/wishlist`, `/cart`, `/profile`); auth guards live in the router's `redirect` callback reading `AuthCubit.state` to enforce public/admin/auth-required route policies.
- Feature packages under `features/<feature>/` follow domain/data/presentation layering: each feature exposes `domain/repositories/*.dart` interfaces consumed by `presentation/cubit/*_cubit.dart` and `data/*_repository.dart` implementations — the DI layer binds interface → implementation.
- Shared cross-cutting concerns (theme, widgets, services, routing helpers) live under `shared/`; generated localization delegates come from `generated/l10n/app_localizations.dart`.
- Static assets (Inter/Montserrat variable fonts, numbered PNGs) are declared under `assets/` and referenced via package paths.
